# gm-social

제미나이와 함께하는 웹페이지 기획 및 콘텐츠 제작
이 튜토리얼의 첫 번째 목표는 **제미나이(Gemini)**라는 똑똑한 AI 비서를 활용하여 웹페이지 제작의 밑그림을 그리는 것입니다. 여러분이 직접 자료를 찾고 디자인을 고민할 필요 없이, 제미나이가 모든 과정을 도와줄 거예요.

참고 영상: https://www.youtube.com/watch?v=389GZE5801M

**1단계: 제미나이 접속 및 주제 정의**

제미나이에게 작업을 시작하기 전에, 먼저 제미나이에 접속해 보세요.

<img width="2121" height="1140" alt="1" src="https://github.com/user-attachments/assets/6ae9ae99-de30-4205-83fe-4036efc18de1" />

그다음 웹페이지의 주제를 명확히 정의하는 것이 가장 중요해요. 

이 강의에서는 **'광명동굴 관광 활성화'**를 주제로 정해 볼게요.

**2단계: 심층 리서치 요청하기**

웹페이지를 만들기 위해서는 먼저 광명동굴에 대한 깊이 있는 정보가 필요해요.  

<img width="2119" height="1142" alt="2" src="https://github.com/user-attachments/assets/48d60e34-c5ed-404a-b224-c77bdfc68f6b" />

이미지를 참고하여 제미나이에게 "광명동굴 관광 활성화를 위한 비전을 소개해달라"고 요청하면, 제미나이가 웹페이지 기획에 필요한 정보를 요약해줍니다.  

<img width="2119" height="1140" alt="3" src="https://github.com/user-attachments/assets/f49c1a97-c06e-4dc6-8314-e3d46cb3d91e" />

제미나이는 단순한 정보 나열을 넘어, 웹사이트 제작에 필요한 구체적인 항목들을 제시해 줍니다. 예를 들어, '관광 활성화를 위한 3가지 방법'처럼 실질적인 조언을 얻을 수 있습니다.

<img width="2119" height="1139" alt="4" src="https://github.com/user-attachments/assets/b4ce429f-5835-4174-9795-83d465dae30c" />

**3단계: 웹페이지 콘텐츠 제작 및 인포그래픽 요청**

이제 제미나이가 제안한 내용을 바탕으로 실제 웹페이지에 들어갈 콘텐츠를 만들어볼 시간입니다. 
제미나이에게 "제안 내용을 바탕으로 세부 보고서를 작성해달라"고 요청하면, 제미나이는 광명동굴의 역사, 현재 상태, 그리고 미래를 위한 전략적 제안까지 아주 상세한 보고서를 작성해 줍니다.

<img width="2117" height="1139" alt="5" src="https://github.com/user-attachments/assets/d085c18c-814c-4c3e-bc20-ab0ad8f44542" />

제미나이가 작성한 상세한 보고서의 일부입니다. 이 보고서는 웹페이지의 훌륭한 텍스트 콘텐츠가 됩니다.

<img width="2117" height="1140" alt="7" src="https://github.com/user-attachments/assets/9e8516ee-c753-4525-963f-fd25aba006e7" />

또한, 복잡한 정보를 시각적으로 쉽게 전달하는 **인포그래픽(Infographic)**도 요청할 수 있습니다.  

<img width="2121" height="1142" alt="9" src="https://github.com/user-attachments/assets/af2a6a6d-16d7-4258-b6bc-8d93915c81ff" />

"이 보고서 내용을 바탕으로 인포그래픽을 만들어달라"고 부탁하면, 제미나이는 방문객 추이와 유형 등을 포함한 시각적인 차트와 함께 인포그래픽의 HTML 소스코드까지 제공해 줍니다.

**4단계: 웹페이지 코드 확인하기**

제미나이는 보고서와 인포그래픽을 만드는 데 그치지 않고,  이 모든 콘텐츠를 담은 웹페이지의 전체 HTML 코드까지 생성해 줍니다. 이 코드를 보면 웹페이지의 뼈대와 디자인 요소들이 어떻게 구성되었는지 알 수 있어요. 우리가 할 일은 이 코드를 복사해서 붙여넣기만 하면 됩니다.

<img width="2122" height="1140" alt="10" src="https://github.com/user-attachments/assets/6d23a480-2baf-4898-912e-91adfca94b2d" />

**🚀 2부: 완성된 코드를 활용한 웹페이지 배포**

이제 제미나이가 만들어준 코드를 실제로 웹페이지로 만들고, 전 세계 사람들이 볼 수 있도록 인터넷에 올리는 방법을 배워볼게요. 이 과정에서는 GitHub라는 도구를 사용합니다.

**5단계: GitHub 저장소 생성 및 코드 업로드**

GitHub이란? 전 세계 개발자들이 코드를 공유하고 협업하는 온라인 공간이에요. 여러분의 웹페이지 코드를 안전하게 보관하고, 웹사이트로 만들어주는 역할을 합니다.

과정:

GitHub 접속: 먼저 GitHub 사이트에 접속해서 회원가입 및 로그인을 해주세요.

<img width="2122" height="1141" alt="11" src="https://github.com/user-attachments/assets/d1c9ba01-6f34-461b-8415-94678f71b746" />

새로운 저장소(Repository) 만들기: 로그인 후, 화면 오른쪽 위에 있는 '+' 버튼을 누르고 **'New repository'**를 클릭합니다.

<img width="2122" height="1141" alt="12" src="https://github.com/user-attachments/assets/83d3631d-5fd3-4bad-9c7b-cd6a766cedd0" />

저장소 이름 설정: 저장소의 이름을 정해주세요. 여기서는 gm-cave처럼 프로젝트를 잘 나타내는 이름이 좋습니다.

<img width="2120" height="1138" alt="14" src="https://github.com/user-attachments/assets/ec0c24c2-eaac-45ea-b619-bbbfc3e2fc5b" />

저장소 생성 완료: 나머지 설정은 그대로 두고 'Create repository' 버튼을 누르면 여러분만의 프로젝트 공간이 만들어집니다.

<img width="2120" height="1140" alt="15" src="https://github.com/user-attachments/assets/f1e26792-2b02-4e77-a8ab-c8ea1e97bd08" />
<img width="2121" height="1142" alt="16" src="https://github.com/user-attachments/assets/73e1ab2c-219d-4e32-9fe9-d5b0b6ae9ca6" />

코드 업로드: 이제 제미나이로부터 받은 HTML 코드를 모두 복사해서, 방금 만든 저장소에 **index.html**이라는 이름의 새 파일로 저장합니다. **index.html**은 웹페이지의 첫 화면을 의미하는 약속된 이름이에요.

<img width="2124" height="1140" alt="17" src="https://github.com/user-attachments/assets/e16041cd-b594-4db4-863b-28fc54215620" />

과정: 1부에서 제미나이가 만들어준 HTML 코드를 모두 복사하세요. 그런 다음, GitHub에 접속해서 새로운 '저장소(Repository)'를 만들고, 복사한 코드를 index.html이라는 이름의 파일로 저장합니다. **index.html**은 웹페이지의 첫 화면을 의미하는 약속된 이름이에요.

**6단계: GitHub Pages로 웹페이지 배포하기**

GitHub Pages란? GitHub에 저장된 여러분의 코드를 실제 웹사이트로 만들어주는 놀라운 기능입니다. 이것만 있으면 웹호스팅 비용 없이 무료로 웹페이지를 배포할 수 있어요.

과정: 여러분이 만든 저장소의 '설정(Settings)' 메뉴로 이동하세요.

**7단계: 최종 결과물 확인**

배포가 완료되면, GitHub Pages가 제공하는 URL을 통해 여러분이 직접 만든 웹페이지에 접속할 수 있습니다.
이제 전 세계 어디서든 이 URL을 통해 여러분의 멋진 광명동굴 웹페이지를 볼 수 있게 된 것입니다!
